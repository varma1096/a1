What's new in Smart Calculator

v 1.6.5

1. 2D graph engine for expressions is improved. Caching technology is implemented to store calculated results. And the expression chart is more sensative to Drag-to-move operations. However, note that plotting expressions is different from plotting data. Expression result at a new value need be calculated anyway. As such if the expression is not simple, user may still experience significant lag.

2. Plot graph for 2-variable and 3-variable implicit functions. Note that only solvable implicit functions can be plotted. Polymonials of degree larger than 3 can be solved but cannot be plotted because the root expressions are very complicated. If plot 2-variable implicit function, at most 4 root expressions can be drawn. For 3-variable implicit function, Calculator may calculate roots for each of the variables and then plot at most 2 roots for every variable. In other words, a single 3-variable implicit function may imply at most 6 to-be-plotted expressions. The whole solving-plotting procedure in this case will take quite long time. Also note that implicit function must be an equation, not assignment, i.e. x**2+y**2=4-z**2 is invalid while x**2+y**2==4-z**2 can be accepted;

3. Change behavior of functions size, zeros and ones. Now if calculate size of a value, not matrix, [] not [0] will return. Also, function zeros and ones return 0 and 1 respectively if input parameter is [];

4. Performance improvement;

5. A number of bug fixes;

v 1.6.2

1. Improved PRINTED math recognition functionality. Now more accurate. Now support addition (95% accuracy), subtraction (95% accuracy), multiplication (90% accuracy), division (90-95% accuracy), mixed fractions and complex fractions (85% accuracy), power (90% accuracy), multi-variable linear equations (80% accuracy, variable name should be x, a, b, c or y), polynomial (80% accuracy, variable name should be x, a, b, c or y), (first level) integral (70% accuracy), summation (80% accuracy) and basic matrix calculation (60% accuracy). Operator can be complex values. Note that handwriting math recognition is still in beta stage.

v 1.6.0

1. Optimized math recognition procedure. Now more flexible, more stable and faster;

2. Support calculation of indefinite integral and add adaptive Gauss-Kronrod algorithm to calculate integral;

5. Result is more accurate. The rounding error now is only 5*10E-48 compared to Octave which can be as large as 10E-17. Only if a value is smaller than 5*10E-48 this app cannot handle (it will be asssumed to be zero);

7. Be able to calculate eigen values and vectors;

9. Add new functions sum_over and product_over;

10. Minor functionality optimized and several bugs fixed;

11. Baidu ads service has been added into this app aiming at users in China mainland. Only if system language is simplified Chinese Baidu ads service is activated. Users in other part of world are not affected. Baidu ads service requires additional permissions including coarse location and phone state to push relative ads to different groups of people. However, neither Baidu nor us store users' private data. Users' privacy is fully protected;
